  <name>Task 1: Install Tremor and create activity feed API</name>
  <files>package.json, src/app/api/admin/activity-feed/route.ts</files>
  <action>
1. Install Tremor and SWR:
   ```bash
   npm install @tremor/react@3.18.7 swr@2.3.3
   ```
   Note: Tremor 3.18.7 includes Recharts as peer dependency - will auto-install.

2. Create activity feed API endpoint at src/app/api/admin/activity-feed/route.ts:

   Cursor-based pagination implementation:
   ```typescript
   // Query params: ?cursor=<id>&limit=10
   // Response: { items: AuditLog[], nextCursor: string | null }
   ```

   Query pattern:
   - Use Prisma cursor pagination: `cursor: { id: cursor }, skip: 1` (skip the cursor item)
   - Default limit: 10 items
   - Order by createdAt DESC (newest first)
   - Include user relation for display name
   - Return nextCursor as last item's ID (or null if no more)

   Response shape:
   ```typescript
   {
     items: Array<{
       id: string;
       action: string;        // CREATE, UPDATE, DELETE, LOGIN, LOGOUT
       entityType: string;    // news, events, documents, users
       entityId: string | null;
       createdAt: string;     // ISO timestamp
       user: { name: string };
     }>;
     nextCursor: string | null;
   }
   ```

   Add Next.js caching: `export const revalidate = 30` (30 seconds for activity feed)

AVOID: Offset-based pagination (performance degrades at scale with OFFSET).
AVOID: Returning full user object - only name needed for display.
  </action>
