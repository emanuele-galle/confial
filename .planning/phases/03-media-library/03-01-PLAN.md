  <action>
**GET /api/media/list:**
- Auth required
- Query params: page (default 1), limit (default 24), folder, tag, search
- Return paginated Media items with total count
- Order by createdAt DESC
- Include presigned URL refresh (7 day expiry)

**POST /api/media/upload:**
- Auth required
- Accept multipart/form-data with multiple files (field name: "files")
- For each file:
  - Validate type (JPEG, PNG, WebP, GIF)
  - Validate size (max 10MB per file)
  - Generate unique filename with UUID
  - Upload to MinIO MEDIA bucket
  - Get image dimensions with Sharp metadata()
  - Create Media record in database
- Return array of created Media objects
- Use after() for Sharp optimization (like existing upload route)

**PATCH /api/media/[id]:**
- Auth required
- Accept JSON body: { folder?, tags?, alt?, focalPointX?, focalPointY? }
- Update Media record
- Return updated Media

**DELETE /api/media/[id]:**
- Auth required
- Delete from MinIO
- Delete from database
- Return success

Use Zod for request validation. Return proper error codes (400, 401, 404, 500).
  </action>

  </verify>
  <done>Media list, upload, and update endpoints work with pagination and filters.</done>

<task type="auto">
  <name>Task 3: Thumbnail + Crop Endpoints</name>
  <files>
src/app/api/media/[id]/thumbnail/route.ts
src/app/api/media/crop/route.ts
  </files>
  <action>
**GET /api/media/[id]/thumbnail:**
- Query params: w (width), h (height), fit (cover|contain|fill, default cover)
- Fetch original from MinIO
- Resize with Sharp: sharp(buffer).resize(w, h, { fit, position: 'centre' })
- If focalPoint set on Media, use position: { left: focalPointX%, top: focalPointY% }
- Return image with correct Content-Type
- Cache-Control: public, max-age=31536000, immutable
- Handle missing media (404)

**POST /api/media/crop:**
- Auth required
- Accept JSON: { mediaId, x, y, width, height, aspectRatio? }
- Coordinates are percentages (0-100)
- Fetch original from MinIO
- Crop with Sharp: .extract({ left, top, width, height })
- Save as new file in MinIO (append "-cropped" to filename)
- Create new Media record linked to original
- Return new Media object with presigned URL

Both endpoints should handle:
- Image format detection (maintain original format)
- Error handling for invalid dimensions
- MinIO connection errors
  </action>
