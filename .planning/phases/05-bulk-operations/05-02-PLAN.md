- Disable close button during processing
- Use EventSource for SSE:
```typescript
const source = new EventSource(`/api/admin/csv/import?execute=true&file=${fileId}`);
source.onmessage = (e) => {
  const data = JSON.parse(e.data);
  if (data.type === 'progress') setProgress(data);
  if (data.type === 'complete') { setPhase('complete'); source.close(); }
  if (data.type === 'error') { setError(data.message); source.close(); }
};
```

Enhance existing batch endpoint with comprehensive audit logging:

**Add audit logging to each batch action (BULK-10):**
- After successful transaction, create AuditLog entry
- Action types: "BATCH_DELETE", "BATCH_PUBLISH", "BATCH_UNPUBLISH", "BATCH_ARCHIVE"
- Include: action, entityType, count of affected items, user who performed action

**Audit log format:**
```typescript
await tx.auditLog.create({
  data: {
    action: `BATCH_${action.toUpperCase()}`,
    entityType: entityType.toUpperCase(),
    entityId: null, // Batch operation, multiple IDs
    userId: session.user.id,
    details: JSON.stringify({
      ids: ids,
      count: success,
      failed: failed,
    }),
    ipAddress: request.headers.get("x-forwarded-for") || "unknown",
  },
});
```

**Move audit log inside transaction:**
- Currently MinIO cleanup is after transaction (correct)
- Audit log should be inside transaction so it rolls back with failed operations

**Verify existing transaction wraps all DB operations (BULK-09, BULK-10):**
